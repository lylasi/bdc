# 同步一致性修復變更摘要

## 本次完成範圍

本次變更聚焦於前端同步鏈路的一致性修復，維持既有 Supabase snapshot / group-level LWW 架構，不調整後端 RPC，也不重做整體同步模型。

## 已完成的關鍵修復

### 1. 自動同步排程語義修正
- 調整 `features/sync/sync.js` 的 auto-sync scheduler。
- 本地資料變更（`data:*`）現在才會進入 debounce + 20 秒最小間隔邏輯。
- `login`、`init-session`、`realtime` 不再污染 `lastLocalChangeAt`，避免被誤當成使用者本地修改。
- `syncInFlight` 期間若有新的本地資料變更，會透過 trailing pass 補跑，不再吞掉中途變更。

### 2. apply / restore 不再把本機再次標髒
- 在 `modules/storage.js` 與 `modules/state.js` 的 persistence helper 中統一支援：
  - `preserveUpdatedAt`
  - `updatedAtOverride`
  - `suppressSyncTouch`
- remote apply 與 backup restore 路徑改為靜默寫回，不再因同步層自己的覆寫又觸發下一輪 auto-sync。

### 3. articles group 與實際持久化資料對齊
- `articleMetas` 已納入 `modules/sync-core.js` 的 snapshot payload。
- `applyArticlesPayload()` 會同步處理：
  - `analyzedArticles`
  - `articleMetas`
- `modules/storage.js` 的 `writeArticleMetasMap()` 已推進 `articlesUpdatedAt`，讓 metadata 真正成為 articles group 的一部分。
- 額外修正刪除 analyzed article 時的細節：清理對應 meta 時不再二次 bump/touch `articles`。

### 4. dictation group 補齊 grading history
- `dictationGradingHistory` 已納入 snapshot。
- `applyDictationPayload()` 會一併處理：
  - 默寫設定
  - AI 批改歷史
- `saveGradingHistory*()` 路徑已與 `dictationUpdatedAt` 對齊，不再出現「有 touch、沒 payload」的空轉同步。

### 5. assistant 同步來源統一到新格式
- `modules/sync-core.js` 已改為使用新版 assistant 資料模型：
  - `assistantConvIndex`
  - `assistant:conv:<id>`
- 不再以 legacy `assistantConversations` 作為主要同步來源。
- snapshot build / apply 已能重建 assistant index 與各會話訊息。

### 6. assistant mutation helper 收斂
- 在 `features/assistant/assistant.js` 已抽出共用 helper：
  - 建立會話
  - 讀取會話訊息
  - 重命名會話
  - 刪除會話
- `features/sync/sync.js` 的 assistant history modal 已改用同一套 helper，避免兩套路徑造成 timestamp / touch 遺漏。

### 7. QA 重複觸發收斂
- `features/qa/qa-storage.js` 中，QA 的 timestamp / touch 已收斂到 `saveQASetsManifest()`。
- `saveQASet()` / `deleteQASet()` 不再各自重複推進 `qaUpdatedAt` 與 `syncTouch('qa')`。

### 8. backup restore 語義修正
- 新增真正的 force-local restore 路徑。
- `features/sync/sync.js` 的還原 UI 已改為呼叫 `restoreSnapshotLocally(payload)`。
- 現在「從備份還原」代表**直接覆寫本機狀態**，不再走 LWW 條件套用。

## 本次實際驗證結果

已透過瀏覽器實際巡檢 / 腳本驗證以下行為：
- `articleMetas` 寫入時會同步推進 `articlesUpdatedAt`
- 刪除 analyzed article 時，對應 meta 會一起刪除，且只觸發一次 `articles` 變更
- `dictationGradingHistory` 寫入時會推進 `dictationUpdatedAt`，且只觸發一次 `dictation` 變更
- assistant helper 會正確更新：
  - `assistantConvIndex`
  - `assistantUpdatedAt`
  - IndexedDB 會話訊息
- assistant 刪除時只觸發一次 `assistant` 變更
- QA 儲存只走單一 manifest/timestamp 路徑
- `buildLocalSnapshot()` 已包含 article metas / grading history / assistant conversations / QA manifest
- `restoreSnapshotLocally()` 已確認會強制套用較舊 payload，而不是被 LWW 擋掉

本次同步一致性修復已完成主要目標：
