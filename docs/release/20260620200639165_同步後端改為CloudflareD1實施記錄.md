# 發佈記錄：同步後端由 Supabase 改為 Cloudflare D1 + Worker

- 日期：2026-06-20
- 類型：架構變更（同步後端替換）
- 認證模式：每人一個通行碼（資料各自獨立隔離）
- 相關文件：
  - 調研 `docs/research/20260620174829159_supabase換本地sqlite可行性分析.md`
  - 計劃 `docs/plan/20260620193423460_supabase轉Cloudflare-D1同步遷移實施計劃.md`

## 動機

Supabase 免費專案頻繁被暫停 / 被提醒「資源占用過高」而封鎖。根因是全量快照同步模型撞上免費層的 egress / realtime 限制 + 7 天閒置暫停。改用 Cloudflare D1（SQLite）+ Worker：按列讀寫計費，單列快照每次同步只算 1 列讀/寫，免費額度幾乎用不到；Workers 無閒置暫停、純配額制不主觀封鎖。

## 程式變更

### 新增
- `cloudflare-worker/schema.sql`：`users`（通行碼雜湊→user_id）+ `snapshots`（單列 JSON 快照）兩張表。
- `cloudflare-worker/src/worker.js`：`/me`、`/version`、`/snapshot`(GET/POST 樂觀鎖) + 通行碼 SHA-256 校驗 + CORS。
- `cloudflare-worker/wrangler.toml`：Worker 與 D1 綁定設定。
- `cloudflare-worker/add-user.mjs`：產生「新增/更新使用者」SQL 的小工具。
- `cloudflare-worker/README.md`：部署與加人說明。
- `modules/sync-cloudflare.js`：前端同步客戶端，導出與舊檔同名接口（`syncNow / auth / subscribeSnapshotChanges / unsubscribeChannel`）。auth 改為通行碼；realtime 改為 45s 輪詢。

### 修改
- `features/sync/sync.js`：import 切到 `sync-cloudflare.js`；`showLoginModal` 改為單一通行碼輸入；移除郵件回跳處理 `handleAuthCallbackIfAny`、`showResetPasswordModal`、`showChangePasswordModal` 及 gear menu 變更密碼項；`onAuthStateChange` 去掉 PASSWORD_RECOVERY 分支。
- `ai-config.example.js` / `ai-config.js`：新增 `SYNC = { endpoint }` 並加入 default export。
- `CLAUDE.md`：更新同步模組描述。

### 保留（回滾後備）
- `modules/sync-supabase.js`：保留不引用，必要時可切回。
- `ai-config` 的 `SUPABASE` 設定：保留。
- `sync-core.js`：合併 / LWW / 套用邏輯完全沿用，未改。

## 待使用者操作（部署）

1. `cd cloudflare-worker && npx wrangler login`
2. `npx wrangler d1 create bdc-sync` → 把 database_id 填入 `wrangler.toml`
3. `npx wrangler d1 execute bdc-sync --remote --file ./schema.sql`
4. `npx wrangler deploy` → 取得 Worker 網址
5. 把 Worker 網址填入 `ai-config.js` 的 `SYNC.endpoint`
6. 為每人加通行碼：`node add-user.mjs "<通行碼>" <user_id> "<備註>"` → 用 `wrangler d1 execute` 套用

## 驗證狀態

- 靜態：`node --check` 通過（sync.js / sync-cloudflare.js / worker.js / add-user.mjs / ai-config*.js）；無殘留舊接口引用。
- 動態：待後端部署 + 填入 endpoint 後，依計劃第 8 節手動驗證（登入、首次上雲、跨裝置合併、錯誤通行碼、離線恢復、既有功能回歸）。
