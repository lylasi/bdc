# 實施計劃：`/version` 輪詢改為「可見性感知 + 自適應退避」

- 時間：2026-06-23 19:39
- 依據研究：`docs/research/20260623192809412_降低version輪詢請求頻率方案.md`（第一階段推薦組合：方案 1 + 方案 2）
- 目標：穩態請求量降 ~85–95%，純前端、不動同步協議。

## 範圍
- 改造 pull 端週期輪詢；**不更動** push 端去抖（`DEBOUNCE_MS/MIN_INTERVAL_MS`）與同步協議。

## 改動點
1. `ai-config.example.js` `SYNC`
   - 新增 `poll: { baseMs, maxMs, pauseWhenHidden, pollOnFocus }`，給預設值與註解。
2. `modules/sync-cloudflare.js`
   - 移除固定 `POLL_MS = 45000`，改 `getPollConfig()` 讀 `SYNC.poll`（含預設與下限保護）。
   - 重寫 `subscribeSnapshotChanges()`：
     - 可變間隔，`base → 2× → … → max` 封頂；偵測到遠端更新或本地資料變更即重置回 base。
     - `pauseWhenHidden`：`document.hidden` 時不排程／暫停。
     - `pollOnFocus`：`visibilitychange` 回前景 / `window.focus` 立即拉一次。
     - 監聽 `bdc:data-changed`（本地活躍 → 重置 base）。
     - `inFlight` 防重入；`stop()` 確實移除所有監聽器（避免洩漏）。

## 行為規格
- 背景分頁：零請求。
- 回前景：立即拉一次並重置回 base。
- 前景閒置：間隔逐步退避到 `maxMs`。
- 本地剛改資料 / 偵測到遠端版本變化：間隔重置回 `baseMs`。

## 預設值
`baseMs=90000`、`maxMs=300000`、`pauseWhenHidden=true`、`pollOnFocus=true`。

## 驗證（手動，DevTools Network 過濾 `/version`）
1. 切背景→停；切回→立即一次。
2. 前景發呆→間隔漸增至封頂。
3. 編輯本地資料→間隔重置回 base；push 去抖不變。
4. 登出→輪詢與所有監聽器停止、無殘留請求。
5. `node --check` 通過；對照 `CLAUDE.md` 同步回歸區。

## 風險
- 純前端、可灰度；唯一行為改變是「背景不再輪詢」（屬預期改善）。不含研究中的方案 3（不犧牲雙裝置即時性的前景行為）。
